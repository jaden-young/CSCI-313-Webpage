Sword

Origional

Variables

  • + bool isAttacking: Tells whether the player is currently attacking. Don't want collision damage to occur just from sword touching dragon.
  • + int damage: Value of damage sword should inflict on an attack
  • + bool Visible: Tells whether the sword is visible (C# Property)

Methods

  • onCollision(object collider): Checks to see if player is currently attacking. If so, and the sword collides with the dragon or destructible objects such as torches, inflict damage.
  • + Visible set method(bool vis): Sets sword visibility (for when player switches weapons)

Actual

We used the sword model found in our Knight from the asset store, with own custom animations. The sword is set as a child object of the Player, keeping its position locked as if the player were holding the sword.

Unity components attached to the sword:

  • Rigidbody - Used so that our collisions register with the physics engine.
  • Box Collider - The swords hit box. This is actually attached to a bone in the armature of the sword model, instead of the parent object, so that the collider follows the sword during the swing animation.
  • Audio Source - Allows the sword to play audio.
  • Animator - Holds information regarding animating the sword. Animations only play when called by the Player Attack script.
  • PlayerAttack Script - Controls swinging the sword and dealing damage. Important components:
    • - bool isAttacking - Tells whether player is currently swinging the sword.
    • - void OnCollisionEnter (Collision) - Called whenever the swords box collider collides with an object.
    • - void Attack () - Causes player to swing sword, playing appropriate sound effects.
    • - void SetIsAttacking () - Sets a private boolean value to determine if the player is currently attacking (so we don't deal damage just from the sword touching the dragon, only when we're swinging the sword). Set by Animation Events in the sword model's animations.
