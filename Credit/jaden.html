Website

When our group first started working on the documentation, we elected to use Google Docs to edit the pages. I was responsible for the initial format of the pages and structure of the site. Also, since the pages were saved as .docx, I was tasked with downloading them, converting them to html files, and submitting them (nothing a little bash magic couldn't handle).

I also made the sidebar navigation for all level's Feature pages. I wish that the school server supported something in the way of templating/includes/etc, I sorely regret doing all of that copy and pasting.

As of 11:45 p.m. 5/5/2016 I made all updates from the original version of all part and scenario pages for level 5. Any page with a hyperlink, I edited. I did everything under the "Actual" heading in parts pages.

Unity Editor

I created the Player by attaching the sword and shield from the knight asset found in the root asset folder to a FirstPersonCharacter prefab from the Unity standard assets.

I configured the hitbox of the dragon by attaching multiple box colliders to different bones of its armature, allowing them to follow the dragon during animations.

I configured the Dragon's flight path using the iTween library.

Code

I wrote every line of code for the custom scripts in level 5. Every item in the Parts section has a script attached to it that is described in its page, excluding the unimplemented Torch part.
